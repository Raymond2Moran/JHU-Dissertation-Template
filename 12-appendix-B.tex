\chapter{Supplementary Optimization Diagnostics} \label{chap:appendix-b}

This appendix records the later log-loss-focused modeling pass that followed the main predictive analysis in Chapter~\ref{chap:protein_results}. The goal of that pass was narrow: train an attention model whose checkpoint selection explicitly targets validation log loss and then test whether a three-seed ensemble of such models should replace the original frozen ensemble. The resulting comparison is useful because it separates two goals that are related but not identical in this thesis: calibrated probability loss on the one hand, and ranking- and planning-oriented surrogate quality on the other.

The frozen surrogate used for inference in Chapter~\ref{chap:protein_results} is a seed-averaged ensemble, so it does not have a single epoch-by-epoch optimization path. For that reason, the optimization curve here is shown for the log-loss-optimized single-attention run, while the ensemble-level comparison is summarized in tables. Figure~\ref{fig:appb_training_loss_curve} tracks train and validation loss across epochs, and Table~\ref{tab:appb_loss_summary} reports the final split-level loss summary at the selected checkpoint. The held-out test split is summarized only once at the selected checkpoint and is not traced across epochs, so that it remains a reporting set rather than a tuning signal.

The main use of Figure~\ref{fig:appb_training_loss_curve} is diagnostic rather than comparative. It shows whether the log-loss-optimized attention model reaches a stable region before early stopping and whether the validation trajectory supports the selected checkpoint. In this appendix, the validation curve is the relevant companion to the training curve because model selection is based on validation behavior, not on the held-out test split.

Table~\ref{tab:appb_loss_summary} complements the optimization curve by giving the final split-level loss summary for the same log-loss-optimized run. Reporting the held-out test loss only once, after checkpoint selection, preserves the logic of the evaluation design used throughout the thesis.

Table~\ref{tab:appb_single_tradeoff} compares the earlier representative single-attention run used in the main predictive table with the later log-loss-optimized single-attention run. The direction of the tradeoff is clear. Optimizing directly for log loss lowers both log loss and Brier on all three key splits, and it slightly improves the external BIOGRID PR-AUC. At the same time, it reduces the in-domain PR-AUC and lowers the pilot \(\rho^2\) values that feed the planning interpretation in Chapter~\ref{chap:protein_results}. In other words, the later run is better calibrated in the loss-based sense, but it is not automatically a stronger surrogate for the later ranking- and planning-based uses of the model.

\input{generated_tables/appendix_b_ensemble_tradeoff.tex}

Table~\ref{tab:appb_ensemble_tradeoff} carries the same comparison to the ensemble level. The three-seed log-loss ensemble improves log loss on all three key splits and improves Brier on two of the three. However, it fails the balanced replacement rule adopted for the thesis: the in-domain test Brier score worsens slightly, the in-domain PR-AUC drops from 0.715 to 0.649, and the pilot \(\rho^2\) on the Intra validation split falls from 0.106 to 0.059. The log-loss-optimized ensemble is therefore retained only as a calibration-oriented contrast, while the original frozen attention ensemble remains the main surrogate used in Chapter~\ref{chap:protein_results}. This is the substantive reason the thesis does not select models by log loss alone.
